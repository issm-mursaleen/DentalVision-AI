# Digital Eye — Oral Cancer Detection

Explainable deep-learning system for binary classification of intraoral clinical
images (Normal vs Abnormal) using ResNet50 transfer learning + Grad-CAM.

The training pipeline mirrors the official `Notebooks/` reference implementation
(authored upstream for the SMART-OM dataset).

## Task

Binary classification:

| Label | Name      | Source folders                                       |
|-------|-----------|------------------------------------------------------|
| 0     | Normal    | `01. Normal`                                         |
| 1     | Abnormal  | `02. Variation from normal`, `03. OPMD`, `04. Oral Cancer` |

Only the `01. Unannotated/<site>/...` subtree is walked per nature folder. The
sibling `02. Region annotation`, `03. Full annotation`, `04. Lesion annotation`
folders contain the same images with overlays drawn on them — including those
would teach the model to detect annotation lines, not pathology.

## End-to-end pipeline

```bash
# 1. Build CSV manifests (once)
python ml/dataset_split.py

# 2. Train the default variant (or pick another from model.py)
python ml/train.py

# 3. (Optional) cross-validate variants for tuning
python ml/kfold.py "ResNet50_mod2_weighted"

# 4. Evaluate on the held-out test split
python ml/test.py

# 5. Serve via FastAPI (auto-loads best_model.pkl from project root)
uvicorn backend.app:app --reload
```

## ML pipeline detail

### `ml/dataset_split.py` — stratified 80/20 split
- Walks `SMART-OM/<nature>/01. Unannotated/<site>/*` for the four nature classes.
- Encodes binary label (Normal=0; Variation/OPMD/Cancer=1).
- Stratifies by `nature + site`; groups with <2 samples are kept entirely in train.
- Writes `train_df_encoded_80_20.csv` and `test_df_encoded_80_20.csv` at the project root.
- Deterministic with `seed=3`.

### `ml/data_loader.py`
- `ImageDataset(df, transform)` — DataFrame-driven (matches the notebook).
- `get_transform()` — `Resize(224) → ToTensor → ImageNet-normalize`. **No augmentation**, matching the notebook.
- `compute_normalized_class_weights(df)` — inverse-frequency weights normalized to sum to 1
  (e.g. `[0.13, 0.87]`). This is what the notebook uses, and it's well-behaved —
  unlike `sklearn.compute_class_weight('balanced')` which produced the
  `[0.29, 32.92]` weights that wrecked our earlier 4-class runs.

### `ml/model.py` — ResNet18 / ResNet50 with variant grammar
Backbone is selected by the prefix in the name (`ResNet18...` or `ResNet50...`).
Default is **ResNet18** — about 2× faster than ResNet50 on Colab and less prone
to overfitting on the ~1.9k training images (the upstream Hyperparameter Tuning
and Testing notebooks themselves use ResNet18). Pass any name to `get_model(...)`.

| Name                              | Backbone | Head (`in` = 512 for R18, 2048 for R50)              | Notes                       |
|-----------------------------------|----------|------------------------------------------------------|-----------------------------|
| `ResNet18`                        | R18      | `Linear(in, 2)`                                      | default                     |
| `ResNet18_mod1_weighted_dp(0.4)`  | R18      | `Linear(in, in)  → ReLU → Dropout(0.4) → Linear(in, 2)` | preferred — wide head        |
| `ResNet18_mod2_weighted`          | R18      | `Linear(in, 256) → ReLU → Dropout(0.5) → Linear(256, 2)` | bottleneck head             |
| `ResNet18_weighted_scratch`       | R18      | `Linear(in, 2)`                                      | from scratch (no ImageNet)  |
| `ResNet50_*`                      | R50      | (same variants)                                      | swap-in if you want R50     |

Tokens parsed from the name:
- `mod1` / `mod2` — head shape (above).
- `dp(<p>)` — dropout probability.
- `weighted` — caller hint to use class-weighted CE loss (no architectural effect).
- `scratch` — drop ImageNet weights, full network trainable.

For pretrained variants, the backbone is frozen except `layer4` and the new `fc` head.

The legacy `get_resnet50_model(...)` symbol is preserved as a thin alias so the
backend and `ml/gradcam.py` continue to work unchanged.

### `ml/train.py` — single-run training (matches Training.ipynb)

| Knob                | Value                                              |
|---------------------|----------------------------------------------------|
| Default `MODEL_LIST`| `ResNet18_weighted_scratch`, `ResNet18_mod2_weighted`, `ResNet18_mod1_weighted_dp(0.4)` (trained in sequence) |
| Loss                | `CrossEntropyLoss(weight=...)` if `weighted` in name; otherwise plain CE |
| Optimizer           | Adam, lr=1e-3, weight_decay=1e-5                   |
| Scheduler           | `ReduceLROnPlateau`, factor=0.1, patience=10       |
| Epochs              | 100 with early stopping, patience=20               |
| Batch size          | 32                                                 |
| Selection metric    | Min validation loss (test split used as val)       |
| Reproducibility     | `seed=3`, `torch.use_deterministic_algorithms(True)`, `CUBLAS_WORKSPACE_CONFIG=:4096:8` |

Saves to `runs/<MODEL_NAME>/`:
- `<MODEL_NAME>_best.pth` — state dict at best val loss
- `<MODEL_NAME>.pth` — final-epoch state dict
- `<MODEL_NAME>_metrics.json` — per-epoch loss/acc/lr
- `<MODEL_NAME>_training_metrics.png` — train/val curves

Also copies the best model to project root as `best_model.pth` + `best_model.pkl`
so the backend just works.

### `ml/kfold.py` — 5-fold CV
Wraps the same training loop in a `KFold(n_splits=5, shuffle=True, random_state=3)`.
Saves per-fold `_best.pth` and aggregated `<MODEL_NAME>_kfold_metrics.json`.

### `ml/test.py` — clinical evaluation
Loads `runs/<MODEL_NAME>/<MODEL_NAME>_best.pth` and reports on the test split:
- Accuracy, Precision, Sensitivity (recall), Specificity, F1, ROC AUC, avg inference time
- `<MODEL_NAME>_results.csv`
- `<MODEL_NAME>_confusion_matrix.png`
- `<MODEL_NAME>_roc_curve.png`

## Backend (`backend/app.py`)

| Method | Path       | Purpose                                                |
|--------|------------|--------------------------------------------------------|
| GET    | `/`        | Ping                                                   |
| GET    | `/health`  | Reports `model_name`, `task_mode`, `class_names`, etc. |
| POST   | `/predict` | Multipart image upload → class + confidence + Grad-CAM |

Boot path:
1. Try to load `best_model.pkl` (full pickled model + metadata: `model_name`,
   `num_classes`, `class_names`, `task_mode`).
2. Otherwise fall back to `best_model.pth` + the default ResNet50 architecture.

This means switching variants in training (e.g. trying `mod2` instead of `mod1`)
**requires no backend changes** — the `.pkl` carries everything the backend needs.

CORS is open for local dev. File validation: type ∈ {jpeg, png, bmp, webp},
size ≤ `MAX_FILE_SIZE_MB`. Error codes: 415, 413, 422, 500.

```bash
pip install -r requirements.txt
uvicorn backend.app:app --reload          # http://127.0.0.1:8000/docs
```

## Frontend (`frontend/src/App.js`)

React 19 + Axios. Reads backend URL from `REACT_APP_API_URL`. Side-by-side
display of original + Grad-CAM. Maps backend HTTP codes (415/413/422/5xx) to
user-friendly errors. Colour-coded diagnosis for backwards-compat with old
class names.

```bash
cd frontend && npm install && npm start   # http://localhost:3000
```

### Earlier training experiments (now replaced)
- First training run used `sklearn.compute_class_weight('balanced')` on 4 classes,
  producing weights `[0.29, 3.46, 4.70, 32.92]`. The 32× spike on Oral Cancer
  destabilised the loss; train accuracy collapsed to 1–10%.
- Second attempt added `WeightedRandomSampler`, `AdamW`, cosine annealing,
  label smoothing, mixup, heavy albumentations, and a `TASK_MODE` switch
  (binary/3class/4class). It improved over baseline but still drifted from
  the official methodology.
- Both have been replaced by the notebook-faithful pipeline below.

### Current pipeline — aligned with `Notebooks/`
Read all four reference notebooks and rebuilt the ML side to match:

- **Binary task** (Normal vs Abnormal) — what the notebooks actually use.
- **Stratified split by `nature + site`** with the singleton-rare-group rule
  (`ml/dataset_split.py`). Produces CSV manifests at the project root.
- **No augmentation** — just `Resize(224) + ToTensor + ImageNet-normalize`,
  matching the notebooks. The earlier `albumentations` pipeline was removed.
- **Normalized inverse-frequency class weights** (sum to 1) — instead of
  the destructive `[0.29, 32.92]` from sklearn.
- **`mod1` / `mod2` head variants** parsed from a single name string
  (`ml/model.py`), with optional `dp(p)` dropout and `scratch` / `weighted`
  flags. Preferred default: `ResNet50_mod1_weighted_dp(0.4)`.
- **Adam (lr=1e-3, wd=1e-5)** + **`ReduceLROnPlateau`** (factor=0.1,
  patience=10), **100 epochs**, early stop=20, model selection by val loss.
  Replaces AdamW + cosine + warmup from the previous attempt.
- **`set_seed` + deterministic CuDNN + `CUBLAS_WORKSPACE_CONFIG`**
  (`ml/train.py`).
- **Multi-file split** (`dataset_split.py`, `data_loader.py`, `model.py`,
  `train.py`, `kfold.py`, `test.py`) so each phase of the notebook pipeline
  has its own runnable entry point.
- **Per-run artifacts** in `runs/<MODEL_NAME>/`. Best model also copied to
  project root as `best_model.pth` + `best_model.pkl` for backend pickup.
- **Backend `/health`** now reports `model_name` and `task_mode` from the
  pickled bundle.

Removed in this pass: `WeightedRandomSampler`, AdamW, cosine annealing, linear
warmup, label smoothing, gradient clipping, AMP, Mixup, `TASK_MODE` remap,
albumentations augmentations, macro-F1 selection. None are part of the
official methodology.

> **Note:** training has **not** been executed — the user runs training themselves.
