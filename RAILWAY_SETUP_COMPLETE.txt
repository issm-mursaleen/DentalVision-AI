🔍 WHAT EACH FILE DOES
================================================================================

Dockerfile.railway
  - Builds the backend API image
  - Python 3.10 + FastAPI + PyTorch + trained models
  - ~1.5GB image size
  - Railway will use this to run your backend

Dockerfile.frontend
  - Builds the frontend React app
  - Node 20 + Vite + React + Tailwind
  - ~200MB image size
  - Railway will use this to serve your web UI

railway.json
  - Configuration for Railway
  - Specifies which Dockerfile to use
  - Sets health check endpoints
  - Defines environment variables

docker-compose.yml
  - For local testing on your machine
  - Runs both services together
  - Use: docker-compose up

requirements-api.txt
  - Production-only dependencies
  - Smaller than requirements.txt
  - Saves ~300MB in image size

verification scripts (ps1/sh)
  - Check if everything is set up correctly
  - Verify model files exist
  - Check git status
  - Validate Dockerfiles

================================================================================
❓ COMMON QUESTIONS
================================================================================

Q: Do I need to set Supabase credentials?
A: No, it's optional. The API works without it. Set them later if you want
   prediction logging.

Q: Can I use a custom domain?
A: Yes! Railway supports custom domains in the service settings.

Q: What if the build fails?
A: Check the logs in Railway dashboard. Usually it's a memory limit on
   free tier. See RAILWAY_DEPLOYMENT.md for solutions.

Q: How do I update the models?
A: Retrain locally, commit models/ folder to git, push, Railway rebuilds.

Q: How much does this cost?
A: $0 on free tier! You get enough for testing. Pay-as-you-go after that.

Q: How long does deployment take?
A: ~5-10 minutes for initial build (depends on Railway's servers).
   Subsequent deployments faster due to caching.
